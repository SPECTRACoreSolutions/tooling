# Stream Deck Developer Research

**What real developers are putting on their Stream Decks for coding**

## 🔍 Research Findings

Based on research of actual developer setups and community best practices, here are the **most intuitive and commonly used buttons** for coding:

---

## 🎯 Top 15 Most Intuitive Buttons (Based on Real Usage)

### **1. IDE Navigation (Most Used)**

**Toggle Terminal** ⭐⭐⭐
- **Why:** Most used button - developers constantly need terminal
- **Shortcut:** `Ctrl+``
- **Usage:** Open/close terminal without leaving keyboard

**Command Palette** ⭐⭐⭐
- **Why:** Access to ALL IDE commands
- **Shortcut:** `Ctrl+Shift+P`
- **Usage:** Quick access to any VS Code/Cursor command

**Quick File Open** ⭐⭐⭐
- **Why:** Navigate files instantly
- **Shortcut:** `Ctrl+P`
- **Usage:** Type filename, jump to file

**Go to Definition** ⭐⭐
- **Why:** Navigate code quickly
- **Shortcut:** `F12`
- **Usage:** Jump to function/class definition

**Find All References** ⭐⭐
- **Why:** See where code is used
- **Shortcut:** `Shift+F12`
- **Usage:** Find all usages of symbol

**Search in Files** ⭐⭐
- **Why:** Search across entire codebase
- **Shortcut:** `Ctrl+Shift+F`
- **Usage:** Find text across all files

---

### **2. Code Editing (Essential)**

**Comment Toggle** ⭐⭐⭐
- **Why:** Frequently used for debugging/testing
- **Shortcut:** `Ctrl+/`
- **Usage:** Comment/uncomment lines instantly

**Format Code** ⭐⭐⭐
- **Why:** Keep code clean automatically
- **Shortcut:** `Shift+Alt+F`
- **Usage:** Auto-format current file

**Split Editor** ⭐⭐
- **Why:** Compare files side-by-side
- **Shortcut:** `Ctrl+\`
- **Usage:** Split editor into multiple panes

---

### **3. Debugging (Critical)**

**Toggle Breakpoint** ⭐⭐⭐
- **Why:** Essential for debugging
- **Shortcut:** `F9`
- **Usage:** Add/remove breakpoint on current line

**Start Debugging** ⭐⭐⭐
- **Why:** Run code in debug mode
- **Shortcut:** `F5`
- **Usage:** Start debugger with current configuration

**Step Over** (Optional)
- **Shortcut:** `F10`
- **Usage:** Step through code line by line

**Step Into** (Optional)
- **Shortcut:** `F11`
- **Usage:** Step into function calls

---

### **4. Git Operations (Very Common)**

**Git Status** ⭐⭐⭐
- **Why:** Check what's changed
- **Shortcut:** `Ctrl+Shift+G` (opens Git panel)
- **Usage:** See modified files, branch status

**Git Commit** ⭐⭐⭐
- **Why:** Save changes frequently
- **Action:** Types `git commit -m "`
- **Usage:** Quick commit workflow

**Git Push** ⭐⭐
- **Why:** Push changes to remote
- **Action:** Types `git push` + Enter
- **Usage:** One-button push

**Git Pull** (Optional)
- **Action:** Types `git pull` + Enter
- **Usage:** Update from remote

---

### **5. Build & Test**

**Run Tests** ⭐⭐⭐
- **Why:** Test code frequently
- **Action:** Types `pytest` + Enter
- **Usage:** Run test suite instantly

**Build/Compile** (Project-specific)
- **Action:** Project build command
- **Usage:** Compile/build project

## 💡 Why These Buttons?

### **Based on Real Developer Feedback:**

1. **Toggle Terminal** - Used constantly, saves reaching for keyboard
2. **Command Palette** - Access to everything, faster than menus
3. **Quick File Open** - Navigate large codebases instantly
4. **Comment Toggle** - Used frequently for debugging/testing
5. **Format Code** - Keep code clean without thinking
6. **Git Status** - Check changes before committing
7. **Git Commit** - Commit frequently, saves typing
8. **Toggle Breakpoint** - Essential debugging tool
9. **Start Debugging** - Run code in debug mode quickly
10. **Run Tests** - Test-driven development workflow
